Angular Data Binding Concepts

Data Binding is the core concept in Angular that allows communication between a component (TypeScript code) and its view (HTML template). It synchronizes the data, making the application dynamic and interactive.

Types of Data Binding

  • One-Way Data Binding: Data flows in a single direction (either from component to view or view to component).
  • Two-Way Data Binding: Data flows in both directions simultaneously.

1-Way Data Binding

In one-way data binding, information passes in one direction. It includes three main types:

1. Interpolation

Definition: Interpolation allows embedding expressions into marked-up text in the template. By default, it uses double curly braces {{ '{{' }} {{ '}}' }}.

Code Example: 
<p>Course Name: {{ '{{' }} courseName {{ '}}' }}</p>
Output in Browser:

Course Name: {{ courseName }}

2. Property Binding

Definition: Property binding allows you to set properties of HTML elements or directives. It uses square brackets [].

Code Example: 
<button [disabled]="isButtonDisabled">Disabled Button</button>
Output in Browser:

3. Event Binding

Definition: Event binding allows you to listen for and respond to user actions like keystrokes, mouse clicks, and touches. It uses parentheses ().

Code Example: 
<button (click)="onButtonClick()">Click Me!</button>
Output in Browser:

Click count: {{ clickCount }}

2-Way Data Binding

Definition: Two-way binding allows for a two-way synchronization between the component architecture and the template. It uses the "bananas in a box" syntax [(ngModel)] and requires FormsModule.

Code Example: 
<input [(ngModel)]="userName" placeholder="Enter name" />
Output in Browser:

Current User: {{ userName }}

Character Count: {{ userName.length }}
